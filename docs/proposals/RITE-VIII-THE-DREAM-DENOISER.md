denoising, physics assistance ONLY — frame interpolation/generation

## The weld (data path)
1. Pleroma AOVs: per-pixel noisy radiance + auxiliary features the
   tracer already knows (albedo, normal, depth) — exported buffers.
   AMENDED post-VIII-0 (adversary review, night-2): the shipped VIII-0
   baseline exports these via a SEPARATE geometry-only primary-hit pass —
   one extra closest-hit traversal per pixel, no light transport, no
   shadow ray, no next-event estimation — not a free read of data the
   beauty pass already produced. This keeps the beauty (accum) path
   byte-identical whether or not AOV export runs (the golden-hash ordeal
   VIII-0 ships). Fusing AOV output into the beauty pass's own existing
   primary hit (removing the second traversal) is a live VIII-1/VIII-2
   optimization option, not done in VIII-0.
2. Training (FORGE-TIME, offline, like transmute): pairs generated by
   OUR integrator — noisy 1-spp in / converged N-spp reference out,
   same (seed, coords), our realms. The net learns only this engine's
   light. Weights artifact = data page, sha-pinned + provenance
   (dataset hash, config) recorded beside it.
3. Inference (RUN-TIME): tiny net (per-pixel fused MLP over radiance +
   aux features, Breda/NRC-class ~6×64 — research/neural-recon.md), a
   compute pass after the integrator, before present. CPU reference
   implementation first; deterministic by construction (fixed
   accumulation order).
4. Present: denoised frame. The noisy frame remains one flag away —
   the pass is a lens, never a dependency.
